# ADR-007: Visual Regression CI Gate

**Status**: Accepted — Accepted in: v0.32.0 via WO-32  
**Phase**: v0.32.0  
**Owner**: Agent D (a11y+visual-regression worktree)

---

## Context

CSS changes to `overlay.css`, `inspector.css`, `banner.css`, and `layer-picker.css` can silently
alter the appearance of shell surfaces. Currently:
- No snapshot baselines exist
- Layout drift from `@layer` cascade changes is only caught by manual review
- Light/dark parity (verified manually in v0.20.0) can silently regress
- New surfaces added in v0.25.x–v0.27.x have no visual baseline

After v0.27.x lands, the shell will have 8+ distinct visual surfaces.
A visual regression gate locks the appearance baseline before v0.28.0+.

---

## Decision

Use Playwright's built-in `toHaveScreenshot()` with a dedicated Chromium project.

### Playwright project config

```javascript
// playwright.config.js — add project
{
  name: "chromium-visual",
  use: {
    browserName: "chromium",
    viewport: { width: 1440, height: 900 },
    deviceScaleFactor: 1,
    colorScheme: "light",
  },
}
```

### Snapshot suite — `tests/visual/shell-visual.spec.js`

Surfaces captured:

| Surface | State | Themes |
|---------|-------|--------|
| Empty state | blank | light + dark |
| Loaded preview | deck loaded, no selection | light + dark |
| Loaded edit — no selection | after mode switch | light + dark |
| Element selected (text entity) | h1 selected | light + dark |
| Element selected — block banner | zoom ≠ 100% | light + dark |
| Floating toolbar | triggered | light + dark |
| Layer picker popup | overlap stack | light + dark |
| Action hint banner | first select | light |

Total: ~15 snapshots.

### Diff thresholds

```javascript
expect(page).toHaveScreenshot({
  maxDiffPixelRatio: 0.01,  // 1% pixel diff tolerance
  threshold: 0.2,           // per-pixel color diff tolerance
});
```

### Snapshot storage

Committed to: `tests/visual/__snapshots__/chromium-visual/`
Generated by: `npm run test:gate-visual -- --update-snapshots`
CI enforces: `npm run test:gate-visual` (no `--update-snapshots`)

### Package changes

```json
// scripts
"test:gate-visual": "playwright test tests/visual/ --project=chromium-visual",
"test:gate-visual:update": "playwright test tests/visual/ --project=chromium-visual --update-snapshots"
```

### Gate integration

`test:gate-visual` runs independently — not blocking Gate-A baseline.
Required to pass before any CSS-touching PR is merged from v0.28.0 onward.

---

## Consequences

**Positive:**
- Snapshots catch layout drift, color regressions, and missing surfaces
- Light/dark parity locked automatically
- `toHaveScreenshot()` is native Playwright — no extra dependencies
- 1% pixel diff tolerance handles sub-pixel font rendering differences across platforms

**Negative:**
- Snapshots are platform-sensitive (Windows vs. Mac renders slightly differently)
- CI must run on a fixed OS (Windows Server 2022 recommended for this project)
- Snapshot updates require intentional `--update-snapshots` run — cannot be automated
- Slow to run (~30s per theme × N surfaces) — must remain a separate CI job
- Any intentional CSS change requires snapshot update (friction, but expected)

---

## Alternatives Considered

1. **Percy / Chromatic** — external services; rejected due to `file://` protocol and offline-first requirement
2. **pixelmatch directly** — considered; `toHaveScreenshot()` preferred as it's maintained in Playwright
3. **CSS unit tests (jest)** — rejected: CSS behavior cannot be fully unit-tested in isolation
4. **Skip visual regression entirely** — rejected: by v0.28.0 the surface area is too large for manual-only review
