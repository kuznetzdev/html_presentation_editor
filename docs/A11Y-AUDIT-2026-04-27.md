### Existing strengths

- Comprehensive landmark structure: single `<h1>` in `<header class="topbar">`,
  three `<aside>` panels each with their own `<h2>`, `<main class="workspace">`,
  `<section class="panel-main">`, `<nav class="mobile-rail">`, plus `<footer>`
  via toast container. Heading hierarchy h1 → h2 → h3 → h4 is monotonic and
  clean (31 headings audited).
- Nearly all interactive controls have an explicit accessible name:
  `aria-label` on icon-only buttons (zoom +/−, panel close ×, broken-asset
  dismiss ×, all 8 selection-frame resize handles 624–666, every floating
  toolbar button 703–846), `<label for=...>` on every inspector input
  (verified across ~40 fields: `widthInput`, `heightInput`, `slideBgColorInput`,
  `borderColorInput`, `transformInput`, `imageSrcInput`, `imageAltInput`,
  `slideTitleOverrideInput`, etc.).
- Live regions instrumented correctly: `aria-live="polite"` + `aria-atomic="true"`
  on `#saveStatePill`, `#zoomLevelLabel`, `#previewLoadingText`, `#toastContainer`;
  `role="status"` on history-budget chip, deck-health badge, selection-summary,
  preview lifecycle pills.
- Strong, theme-aware focus ring: `--focus-ring-width: 3px` + dedicated
  `--focus-ring-color` token, applied via single `:focus-visible` rule in
  `editor/styles/base.css:174–188` to `button, input, select, textarea, a,
  [tabindex], .layer-row, .quick-palette button, .context-menu button,
  .layer-picker button`. v2.0.22 bumped focus alpha from 0.18 → 0.32.
- Reduced-motion support is broad and explicit: `editor/styles/responsive.css:472–503`
  zeroes `transition`, `animation`, and `transform` on 14 components when
  `prefers-reduced-motion: reduce`; `base.css:103` cancels the v2.0.22 active
  scale-down feedback; `onboarding.css:35` and `onboarding.css:54` disable two
  more onboarding-specific animations.
- Roving-tabindex pattern correctly implemented on slide rail (verified
  via `tests/a11y/keyboard-nav.spec.js` P0-08): exactly one `.slide-item`
  has `tabindex="0"`; ArrowDown/ArrowUp shift it; Alt+ArrowDown reorders.
- Russian copy preserved in all `aria-label` values (P0-05 test enforces
  Cyrillic regex on every slide-item label).
- All token-driven contrast pairs **PASS WCAG AA** (computed below): every
  light-theme combination ≥ 4.70:1, every dark-theme combination ≥ 5.94:1.
  v2.0.16 / A11Y-001 fix raised `--shell-text-muted` light-theme alpha from
  0.6 → 0.78 — math confirms 5.62:1 on white (was 3.43:1).

### Top 5 priority fixes

1. **CRITICAL — Hardcoded badge fg colors fail dark-theme contrast.**
   `editor/styles/base.css:124–146` (`.deck-health-badge-low/medium/high/severe`)
   and `:148–168` (`.experimental-badge`) hardcode foreground `rgb(138,95,0)`,
   `rgb(24,110,73)`, `rgb(164,46,8)`, `rgb(164,22,34)` over translucent rgba
   backgrounds. In light theme the composite ratios are 5.06:1 – 5.82:1
   (PASS). In dark theme the composites collapse to **1.86:1 – 2.14:1**
   (FAIL — needs ≥ 4.5:1). The `deck-health-badge` is permanently visible
   in topbar after deck import. Fix: add `[data-theme="dark"]` rebinds with
   light foreground colors against the same translucent backgrounds.
2. **CRITICAL — Four shell modals lack `role="dialog"` + `aria-modal="true"`.**
   `#openHtmlModal`, `#htmlEditorModal`, `#videoInsertModal`, `#shortcutsModal`
   (lines 1678, 1775, 1813, 1879) are plain `<div class="modal" aria-hidden="true">`
   without dialog semantics. Screen readers will announce them as generic
   regions instead of announcing modal context (NVDA: "dialog" + heading;
   without role they get nothing). Compare to `#layerPicker` line 1943 (correct)
   and `#importReportModal` (correct, set in `import-report-modal.js:51-53`).
   Fix: add `role="dialog" aria-modal="true" aria-labelledby="<h3-id>"` to all
   four. Add `id` to the heading where missing (only `htmlEditorModalTitle`
   exists today).
3. **MAJOR — Inspector text-align buttons have no accessible name.**
   `editor/presentation-editor.html:1222–1224` are `<button data-align="left|center|right" title="…">←</button>`. Screen readers will speak the visible
   character "←" / "↔" / "→" only — `title` is not reliably surfaced (browser
   hover, not AT). The floating-toolbar versions at 810–812 already have
   `aria-label` — apply the same pattern. Same issue for `bold`/`italic`/`underline`
   buttons 1213/1216/1219 (children `<strong>B</strong>`, `<em>I</em>`,
   `<span>U</span>` — accessible name becomes "B"/"I"/"U" which is opaque).
4. **MAJOR — No skip-link / bypass-blocks landmark target.** First Tab from a
   freshly-loaded shell traverses the topbar (~10 buttons: theme, undo, redo,
   open, present, export-html, export-pptx, overflow ⋯, mode toggle x2) before
   reaching the workspace. WCAG 2.4.1 (Level A) requires a "skip to main content"
   link as the first focusable element. Implementation: add
   `<a class="skip-link" href="#workspace">К рабочей области</a>` immediately
   after `<body>`, with CSS that visually hides it until `:focus-visible`.
5. **MAJOR — `aria-hidden="true"` on closed modal containers blocks every
   focusable child.** All five shell modals plus `#contextMenu`,
   `#layerPicker`, `#topbarOverflowMenu`, `#slideTemplateBar`, `#quickPalette`
   sit in the DOM with `aria-hidden="true"` while closed. This is correct by
   itself, but combined with the absence of `role="dialog"` + `inert` (see
   #2) the elements still receive Tab focus on some browsers (Chrome
   `aria-hidden` does **not** remove from tab order — it only hides from AT
   tree). Verify with `tabindex` audit; if any focusable descendants are
   reachable while modal closed, add `inert` attribute or programmatic
   `tabindex="-1"` management. (gate-a11y P0-05 modal-trap test passes for
   open state but does not assert closed state.)

## Reduced-Motion Audit

`prefers-reduced-motion: reduce` coverage. Any animation > ~200ms left
unchecked is flagged.

| File:line | Animation | Duration | Disabled on reduce? |
|---|---|---|---|
| `base.css:99–101` | `button:active { transform: scale(0.98) }` | instant | yes (103–107) |
| `responsive.css:472–503` | 14 components — buttons, slide-item, floating-toolbar, context-menu, quick-palette, toast, spinner, section-toggle, layer-row, layer-tree-node summary, layer-label-input, left-pane-resizer, layers-region, import-report-modal, icons | various | yes — `transition: none !important; animation: none !important; transform: none !important` |
| `onboarding.css:35` | onboarding bubble pulse | unknown | yes |
| `onboarding.css:54` | onboarding entry animation | unknown | yes |
| `tokens.css:91-93,126-128` | `--motion-fast: 160ms`, `--motion-medium: 180ms`, `--motion-emphasis: 280ms` | up to 280ms | not directly — depends on consumer respecting media query |
| `responsive.css:505–525` | `@keyframes spin` (loading spinner) | infinite | spinner included in :485 disable list — yes |

**Gap:** No global `* { animation-duration: 0.01ms }` fallback. Components
that consume `--motion-emphasis` (280ms) without being in the explicit list
on `responsive.css:479–498` will still animate. Audit recommends adding a
catch-all rule:
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```
Then keep the existing component-specific overrides for the few cases needing
`transform: none` specifically.

## Recommendations for Track D Phase D4 (50+ a11y tests, currently 27)

The current suite covers axe-core scans (3 workflow states), 7 token-pair
contrast assertions per theme (14 total), and 6 keyboard-nav tests. To
expand from 27 → 50+ while addressing this audit's findings:

1. **Hardcoded-color contrast scan (5 new tests).** Add a programmatic
   sweep that walks every element with class matching
   `^(deck-health-badge|experimental-badge|.*-badge)` — read computed
   `color` and `background-color` via `page.evaluate` after themes flip,
   composite alpha against parent, assert ≥ 4.5:1. This catches P-01 + future
   regressions of similar hardcoded styles.
2. **Modal semantics gate (4–6 new tests).** For each shell modal
   (`openHtmlModal`, `htmlEditorModal`, `videoInsertModal`, `shortcutsModal`,
   `importReportModal`, `layerPicker`) assert open state has
   `role="dialog"` + `aria-modal="true"` + `aria-labelledby` pointing at a
   non-empty `<h*>` — direct fix for O-02.
3. **Skip-link / first-focusable test (2 new tests).** Press Tab once on
   fresh load — assert `document.activeElement` is the skip-link, then
   second Tab lands on the first topbar button. Drives O-01.
4. **Accessible-name coverage sweep (1 parameterized test).** Walk every
   `button` and `input` in the shell, compute accessible name via the
   ARIA-name algorithm (axe internal `accessible-text` API), assert
   non-empty and not equal to a single Unicode glyph. Catches O-03 and O-04
   class issues.
5. **Reduced-motion enforcement test (3 new tests).** Use
   `page.emulateMedia({ reducedMotion: 'reduce' })`, then trigger
   slide-create / toast-show / modal-open and assert
   `getComputedStyle(el).animationDuration === '0s'` (or near-zero).
   Catches gaps left by the component-specific `responsive.css` list.
6. **Live-region announcement timing (2 new tests).** Trigger zoom +/−
   and verify `#zoomLevelLabel` text actually changes within ≤ 200ms;
   trigger save and verify `#saveStatePill` text changes; both must remain
   `aria-live="polite"`.
7. **Iframe boundary tests (2 new tests).** Confirm Tab does not steal
   focus into `#previewFrame` from the shell unexpectedly; confirm Escape
   from inside the iframe returns focus to the last shell-side opener.
