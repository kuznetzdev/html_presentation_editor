Reveal-style структура без внешней библиотеки

Этот файл специально написан в совместимом стиле: .reveal .slides > section. Так удобно тестировать редактор на deck-структуре, похожей на reveal.js.

Вертикальная вложенность — часть 1

Зачем

Проверка nested slide groups и корректной навигации по секциям.

Риск

Редактору сложнее определять активный слайд и порядок при nested markup.

Вертикальная вложенность — часть 2

  1. Определить группу.
  2. Выделить текущий внутренний слайд.
  3. Сохранить корректную активацию и экспорт.

Что это даёт

  • Похожий на реальный framework deck-маркап.
  • Хороший regression-файл для slide model и runtime metadata.
  • Проверка поведения при nested sections.
