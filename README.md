# HTML Presentation Editor — rebuilt v4

## Что это
Локальный редактор HTML-презентаций с живым preview в `iframe`.

Ключевой принцип:
- preview старается вести себя как обычное открытие HTML в браузере;
- редактирование идёт поверх живого DOM через bridge;
- экспортируется чистый HTML без служебных атрибутов редактора.

## Основные возможности
- загрузка HTML из файла или текста;
- preview / edit режимы;
- список слайдов с быстрым переходом;
- выбор почти любого элемента внутри слайда;
- редактирование текста, картинок, контейнеров, iframe-видео;
- добавление новых элементов: текст, картинка, видео, блок, раскладки;
- изменение размеров, позиции, оформления и атрибутов;
- контекстное меню по правому клику;
- плавающая панель быстрых действий;
- undo / redo;
- autosave в `localStorage`;
- вставка изображения через `Ctrl+V` и drag-and-drop;
- copy / paste style;
- экспорт итогового HTML.

## Как открыть
1. Открой файл `presentation-editor-rebuilt-v4-commented.html` в браузере.
2. Нажми **Открыть HTML**.
3. Выбери локальный `.html` файл или вставь HTML вручную.

## Режимы
### Превью
- iframe показывает презентацию почти как обычный документ;
- навигация презентации и её скрипты стараются работать как есть;
- редактор не включает `contenteditable` и не мутирует DOM без необходимости.

### Редактирование
- можно выбирать элементы кликом внутри iframe;
- двойной клик по тексту включает редактирование;
- справа появляется инспектор, а около элемента — floating toolbar.

## Горячие клавиши
- `Ctrl/Cmd + Z` — undo
- `Ctrl/Cmd + Y` или `Ctrl/Cmd + Shift + Z` — redo
- `Ctrl/Cmd + D` — duplicate element
- `Ctrl/Cmd + Shift + C` — copy style
- `Ctrl/Cmd + Shift + V` — paste style
- `Ctrl/Cmd + F` — поиск элемента на активном слайде
- `Delete / Backspace` — удалить выбранный элемент
- `Escape` — закрыть контекстное меню / модалку

## Что умеет контекстное меню
Меню адаптируется к типу элемента.

### Для текста
- редактировать HTML;
- редактировать текст;
- копировать / вырезать текст;
- вставить как текст;
- преобразовать в `H2` / `H3`.

### Для изображения
- заменить изображение;
- копировать URL;
- открыть URL;
- сбросить размеры;
- вписать по ширине;
- повернуть / отразить через инспектор.

### Для контейнера
- добавить дочерний текст;
- добавить дочернее изображение;
- обернуть в новый `div`.

## Что делает floating toolbar
- удалить;
- дублировать;
- копировать / вставить стиль;
- редактировать текст;
- менять цвет и размер текста;
- заменить картинку;
- копировать URL картинки;
- менять URL видео.

Панель можно:
- перетаскивать;
- сворачивать;
- она запоминает позицию в рамках сессии.

## Автосохранение
Редактор периодически сериализует проект в `localStorage`.
После перезагрузки предлагает восстановить последнюю сохранённую версию.

## Экспорт
Кнопка **Экспортировать HTML**:
- берёт `modelDoc`;
- убирает `data-editor-*`, bridge-скрипт и helper-артефакты;
- скачивает итоговый `presentation-edited.html`.

## Ограничения
- если презентация завязана на сложный пользовательский JS, после агрессивных структурных правок её логика может частично сломаться;
- локальные относительные ресурсы (`./img.png`, `./style.css`) надёжнее работают при запуске через `localhost`, а не через `file://`;
- editor не обещает полную совместимость со всеми нестандартными deck framework'ами.

## Что внутри по архитектуре
- `modelDoc` — модель документа для истории и экспорта;
- `iframe` — живое превью;
- `bridge` — код внутри iframe для выбора элементов и применения изменений;
- `postMessage` — транспорт между shell и preview;
- `MutationObserver` — подхват новых узлов в preview;
- `localStorage` — autosave;
- history snapshots — undo / redo.

## Где смотреть подробности
Для разбора кода по блокам используй HTML-файл `presentation-editor-rebuilt-v4-commented.html`: в нём добавлены встроенные комментарии по секциям shell/UI/bridge/history/export.
