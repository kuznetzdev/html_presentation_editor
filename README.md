# HTML Presentation Editor — README

## Что это

`presentation-editor-rebuilt-v2.html` — это локальное мини‑приложение для визуального редактирования HTML‑презентаций.

Редактор предназначен для двух режимов:

- **Превью** — показывает загруженную презентацию в `iframe` максимально близко к обычному открытию HTML в браузере.
- **Редактирование** — позволяет аккуратно менять содержимое и стили элементов, не сводя работу к правке исходника вручную.

---

## Что уже умеет текущая версия

### Загрузка и превью
- загрузка HTML из файла;
- загрузка HTML из вставленного текста;
- указание `Base URL` для относительных `src` / `href`;
- загрузка превью через `Blob` URL в `iframe`;
- сохранение работы скриптов презентации, если они находятся внутри исходного HTML.

### Работа со слайдами
- определение слайдов через reveal.js API или fallback‑селекторы;
- список слайдов слева;
- переход к слайду;
- изменение порядка слайдов;
- обновление списка после изменений.

### Редактирование
- выбор почти любого элемента на слайде;
- точечное редактирование текста через `contenteditable`;
- редактирование атрибутов `id`, `class`, `src`, `alt`;
- изменение стилей через инспектор:
  - цвет текста;
  - размер шрифта;
  - выравнивание;
  - фон;
  - границы;
  - `margin` / `padding`;
  - `width` / `height`;
  - `position`, `left`, `top`, `z-index`, `display`.

### Операции с элементами
- удалить;
- дублировать;
- сдвинуть вверх/вниз по DOM;
- сбросить inline‑стили;
- отредактировать HTML элемента;
- отредактировать HTML активного слайда.

### Вставка новых элементов
- текстовый блок;
- изображение;
- видео (YouTube / Vimeo через embed URL);
- custom HTML.

### UX-функции
- контекстное меню по правому клику;
- двойной клик по тексту для быстрого входа в редактирование;
- плавающая мини‑панель;
- Undo / Redo;
- autosave в `localStorage`;
- вставка изображения через `Ctrl+V`;
- дублирование через `Ctrl+D`;
- drag-and-drop изображения в зону превью;
- предупреждение при закрытии вкладки с несохранёнными изменениями.

### Экспорт
- экспорт полного HTML‑документа;
- сохранение `doctype`;
- очистка служебных `data-editor-*`, bridge‑скрипта, helper‑стилей и `contenteditable`.

## Как пользоваться

### 1. Открыть редактор
Открой файл:

```html
presentation-editor-rebuilt-v2.html
```

в современном браузере.

---

### 2. Загрузить презентацию
Нажми **«Открыть HTML»** и выбери один из вариантов:

- загрузка `.html` файла;
- вставка полного HTML в текстовое поле.

Если у презентации есть относительные пути к ресурсам (`./img/...`, `./styles/...`), укажи **Base URL**.

---

### 3. Переключать режимы
В верхней панели:

- **Превью** — для навигации и просмотра;
- **Редактирование** — для выбора и изменения элементов.

---

### 4. Выбрать элемент
В режиме редактирования:

- клик — выбрать элемент;
- двойной клик по тексту — начать редактирование текста;
- правый клик — открыть контекстное меню.

---

### 5. Редактировать
Справа в инспекторе можно:

- изменить текстовые стили;
- изменить размеры и позицию;
- изменить фон, границы, отступы;
- отредактировать `src` / `alt` картинки;
- вставить текст / картинку / видео / custom HTML;
- дублировать, удалить или переставить элемент.

---

### 6. Вставить изображение
Способы:

- кнопка **«Картинка»** в инспекторе;
- `Ctrl+V`, если в буфере картинка;
- drag-and-drop файла в область превью;
- кнопка **«Заменить изображение»** для выбранного `<img>`.

---

### 7. Экспортировать
Нажми **«Экспортировать HTML»**.

Скачается файл:

```html
presentation-edited.html
```

---

## Горячие клавиши

- `Ctrl+Z` / `Cmd+Z` — undo
- `Ctrl+Y` / `Cmd+Shift+Z` — redo
- `Ctrl+D` / `Cmd+D` — дублировать выделенный элемент
- `Delete` / `Backspace` — удалить выделенный элемент (если не редактируется текст)
- `Esc` — выйти из `contenteditable` у выбранного элемента

## Ограничения

### 1. Не гарантируется сохранение сложной логики презентации
Если пользователь ломает критическую DOM‑структуру, классы или обёртки, от которых зависит reveal.js или кастомная логика презентации, редактор **не гарантирует**, что анимации, переходы и скрипты останутся рабочими.

### 2. Локальные ресурсы и `file://`
Если презентация загружается локально и использует относительные пути, в некоторых браузерах могут быть ограничения, связанные с `file:` origin. В таких случаях лучше:
- запускать редактор через локальный сервер (`localhost`);
- указывать `Base URL`;
- или использовать самодостаточный HTML.

### 3. Текущий MVP не имеет мышиных resize-handles и полноценного drag-move элементов
Сейчас размеры и позиция меняются через инспектор.

## Что важно понимать по архитектуре

Редактор **разделяет**:

1. **modelDoc** — редактируемую модель HTML в основном окне;
2. **preview iframe** — живой runtime презентации;
3. **bridge** — скрипт‑посредник внутри iframe;
4. **UI редактора** — панели, модалки, история, autosave, экспорт.

Это сделано, чтобы:
- не редактировать preview напрямую “вслепую”;
- не терять состояние для экспорта;
- не завязываться на один конкретный движок презентаций.

## Что ещё запланировано

Высокий приоритет:
- умное контекстное меню с группировкой, иконками и действиями по типу элемента;
- сворачиваемые секции инспектора;
- toast‑уведомления;
- перетаскиваемая плавающая панель;
- копирование/вставка стиля;
- более умное позиционирование floating toolbar.

Средний приоритет:
- layout presets;
- richer media actions (копировать URL картинки, fit/reset size, rotate/crop light);
- drag from palette to slide.

Низкий приоритет:
- тёмная тема;
- история с миниатюрами;
- поиск элементов.
